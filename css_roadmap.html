What Are CSS Specifications & Why Read Them?

The authoritative source ‚Äî not MDN, not blogs

A CSS specification is the formal document that defines how CSS must behave. Browser engineers (Chrome/V8, Firefox/Gecko, Safari/WebKit) implement these specs. If there's a conflict between MDN and a spec, the spec is always correct ‚Äî MDN may have a lag or an error.

üìå Official Source The canonical home for all CSS specs is https://www.w3.org/Style/CSS/. The live index of all work-in-progress and completed specs is at https://www.w3.org/Style/CSS/current-work. This page is maintained by the CSS Working Group (CSSWG) itself.

Modules vs Levels ‚Äî The Modular Architecture of CSS

Why there's no single CSS3 spec file

Before CSS3, everything was in one giant document. This was a bottleneck ‚Äî the whole spec had to advance together. The CSSWG fixed this by splitting CSS into independent modules, each advancing at its own pace.

Modules

Each module covers a specific area of CSS. Examples: CSS Flexbox Module Level 1, CSS Grid Layout Module Level 2, CSS Color Level 5. You can think of a module as a chapter in a book, except each chapter gets published and versioned independently.

Levels

A "level" is a module's version number ‚Äî but levels don't just add features, they supersede and extend previous levels.

LevelMeansExample
Level 1First specification for this topic, sometimes corresponding to CSS1 era.CSS Selectors Level 1 = original CSS1 selectors
Level 2Extends Level 1. Redefines and adds features.CSS Selectors Level 2 = added attribute selectors
Level 3Extends Level 2 (or starts fresh if no prior level covered this topic).CSS Selectors Level 3 = added :nth-child(), :not(), etc.
Level 4Latest iteration; some Level 4 features are already shipping in browsers.CSS Selectors Level 4 = :is(), :where(), :has()
üìå Verified Fact Some modules start at Level 3 if they extend something that was in CSS2.1 (e.g., "Backgrounds and Borders Level 3" ‚Äî there's no Level 1 or 2 because borders were part of the monolithic CSS2.1 spec). Source: Russ Max Design / W3C CSS specs page.

CSS Snapshots

Because modules advance separately, it can be hard to know "what is CSS right now?" That's what CSS Snapshots are for. The snapshot lists all modules that are stable enough to be considered the current state of CSS. The latest is CSS Snapshot 2025 at https://www.w3.org/TR/css-2025/.

üí° Pro Tip When someone asks "does this property exist in CSS?" the best authoritative check is the CSS Snapshot properties index at https://www.w3.org/TR/css/#properties. This lists every stable property with its module source.

The Property Definition Table

Every property in a spec has a definition table. Here's how to read it:

Name: grid-template-columns Value: none | <track-list> | <auto-track-list> | subgrid <line-name-list>? Initial: none ‚Üê Default value before any CSS is applied Applies to: grid containers ‚Üê Only works on display:grid elements Inherited: no ‚Üê Children do NOT inherit this Percentages: refer to inline size of the grid container Computed value: as specified, with lengths made absolute Animation type: simple list of length, percentage, or calc
üí° Reading Tip The property definition table is the MOST IMPORTANT part of any property's spec entry. It answers: what are valid values? what's the default? does it inherit? what elements does it work on? Read this table before anything else about a property.

Reading the Formal Syntax Grammar

The most intimidating part ‚Äî demystified completely

CSS specs define every property's valid values using a formal grammar notation. Once you can read this, you can determine exactly what any property accepts ‚Äî no guessing.

The Value Definition Syntax (Multipliers)

(space) ‚Äî juxtaposition = all components must appear in order. bold 12px serif
&&All components must appear, in any order. A && B = A then B, or B then A
||One or more must appear, in any order. A || B = A, B, or both
|Exactly one of the alternatives. A | B | C = pick one
[ ]Grouping ‚Äî like parentheses in math

The Value Definition Syntax (Multipliers)

?Optional ‚Äî zero or one times
*Zero or more times
+One or more times
{A,B}Between A and B times
#One or more, comma-separated
!Group must produce at least one value

Data Types

<length>Any length value: px, em, rem, vw, %...
<number>Any real number: 0.5, 1, -3...
<integer>Whole numbers only
<color>Any color value ‚Äî see CSS Color Module for full grammar
<image>url(), gradient(), etc.
<string>Quoted string: "hello"
<ident>Any CSS identifier (unquoted keyword)

Worked Example: Decoding a Real Grammar

Let's decode the grammar for the background shorthand from the CSS Backgrounds spec:

background: [ <bg-layer>, ]* <final-bg-layer>

This reads as: zero or more <bg-layer> values each followed by a comma, then exactly one <final-bg-layer>. That's why multiple backgrounds are comma-separated, and why the last one can have a background-color.

border-radius: <length-percentage>{1,4} [ / <length-percentage>{1,4} ]?

Reading: 1 to 4 length/percentage values, optionally followed by a / and another 1 to 4 values. The slash separates horizontal and vertical radii ‚Äî this is why border-radius: 10px / 20px creates elliptical corners.

üìå Where This Grammar Is Defined The formal Value Definition Syntax is specified in CSS Values and Units Module Level 4 ‚Äî w3.org/TR/css-values-4/. Read this spec once and you'll be able to decode every other spec's property definitions.

Workflow: How to Research a Property Like a Pro

1

Find the defining module

Go to drafts.csswg.org/indexes/#properties ‚Üí Ctrl+F the property name ‚Üí note which module defines it.

2

Check the module's status

Go to the module page ‚Üí check "Status of this Document" ‚Üí CR or REC = use in production.

3

Read the property definition table

Check: initial value, applies-to, inherited, computed value, animation type. This answers 80% of common questions.

4

Decode the formal grammar

Use Section 6 of this guide to parse the value definition. Write out 3 examples yourself to verify understanding.

5

Cross-check browser support

Go to caniuse.com + wpt.fyi. A spec can say something is required but browsers might not have shipped it yet.

6

Verify against MDN

MDN is a good sanity-check for your reading. If MDN says something different from the spec, the spec is correct ‚Äî but also check if MDN might be referencing a newer spec version.
